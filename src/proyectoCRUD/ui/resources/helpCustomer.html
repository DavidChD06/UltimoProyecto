Bank Customer Management Application

This application allows the managment of all customers in the database. This includes the creation, deletion and updating

1. General Structure

The application is developed in JavaFX following a client-server pattern, where the graphical interface communicates with a REST backend via the CustomerRESTClient class.

The main view displays a table (TableView) with all the customers registered in the database.

2. Customer Table

The table displays the following information for each customer:

  • ID: Unique identifier of the customer (read-only).
  • First name: (editable).
  • Middle initial: (editable).
  • Last name: (editable).
  • Street: (editable).
  • City: (editable).
  • State: (editable).
  • Zip: (editable).
  • Phone: (editable).
  • Email: (editable).
  • Password: (editable).

3. Data Editing

All fields except the field "ID" can be edited, each field has its own validation:

  • First name: Name must not be too long nor null.
  • Middle initial: Middle initial must contain just one letter and not be null .
  • Last name: Surname must not be too long nor null.
  • Street: Street must not be too long nor null.
  • City: City must not be too long nor null.
  • State: State must not be too long nor null.
  • Zip: Zip must contain just numbers, only 5 digits and not be null.
  • Phone: Phone must just contain numbers, have a set length, and not be null.
  • Email: Email must follow the standart form of an email and not be null.
  • Password: Password must be between 5 and 30 characters, contain at least one number and one upper case.

4. Creating New Customers

Account creation is done via a button:

  • When pressed, a new entry in the table is created and selected.
  • All the fields except the id are empty.
  • The user can leave empty fields at the moment of creating.

5. Update and Persistence

For existing customers, all modifications made from the table is automatically sent to the backend via REST calls.

This ensures that changes are reflected immediately in the database.

6. Data Refresh

The Refresh button requests all customers from the server again and reloads the table with the updated data.

This mechanism ensures that the view is synchronized with the actual server state.

7. Deleting

A customer can only be deleted if it has no associated accounts. Before proceeding, the application requests confirmation from the user.

9. Conclusion

The application provides complete and secure bank customer management, combining interface validations, automatic data updates, and efficient communication with the REST backend.
